Terminal relaxation

If the variable OutTermRelaxPathways is set to yes via the resource file, information about terminal relaxation pathways of each chain are written in an output file. The default file name is TermRelax.dat (or, .txt). The default file name can be changed via the variable OutTermRelaxFNM (The appropriate extension will be added to the filename by the code). 

If the variable Add_header is not set to no, the output file will have a header line detailing the output. The fifth column in the output codes for the relaxation pathway with an integer. Depending on the relaxation pathways, the meaning of the entries in the other columns can be different and are detailed below:

  1. Index i: The first column simply states the index of the polymer in the ensemble. This is useful for blends where molar masses of different components overlap. Since the indices are given based on the appearance of the polymers in the inputfile, this variable can be used to separate polymers belonging to different components sharing the same relaxation pathway.

  2. Weight fraction wi: The second column gives the weight fraction carried by this particular "chain" (remember a "chain" may represent all the chains having molar masses in a particular molar mass range). 

  3. Molar mass Mi: The third column gives the mass of the chain.

  4. Relaxation time τrelax : For unentangled chains, this time is set as the Rouse time τR . For chains whose terminal relaxation involves reptation this time is set to the longest reptation time τd,0 . Chains in the highest molar mass end may become disentangled in the fat tube and relax with the timescale of being disentangled. For this chains, the time τdis in this entry are the time when the chains first become disentangled. Since the disentanglement is checked after performing the arm retraction during the time interval, some chains can be assinged terminal relaxation via contour length fluctuation in the fat tube and the time for those chains are given when all stress are relaxed via CLF τCLF  .
